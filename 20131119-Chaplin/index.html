Framework


Application


Initializes everything, the bootstrapper of the application, and an extension point for key parts of the architecture.

Router


Handles URL changes and reversals. Facilitates mapping URLs to controller actions based on a user-defined configuration file. It does no direct action apart from notifying the dispatcher of such a change.

Dispatcher


Loads new controllers & disposes old ones. Listens for routing events. Calls methods of controllers.

Controller


Initializes models and views as properties for simple memory management. Typically represents one screen of an application. Controller actions are called by the dispatcher.

View / Template


Handles the logic that drives the user interface such as responding to DOM events and mapping data from the model to the template.  HTML templates are rendered by the View.  

Layout


The top-level application view. Handles all document links and pings Router if it is internal. Manages view regions. Top-level DOM events on documentor bodyshould be registered in the Layout.

COmposer


Persists composed controller properties between screens. Identifies which regions of the new screen should not change. Increases performance and saves memory.

Model


Stores and syncs data. Contains any logic necessary to retrieve from and send data to the source.
